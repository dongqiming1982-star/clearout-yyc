# Lead Photos

Clearout YYC lead photos use Supabase Storage, not Vercel, GitHub, Postgres blobs, or local disk.

## Storage bucket

Bucket: `lead-photos`

Recommended settings:

- Private bucket
- File size limit: 1 MB
- Allowed MIME types: `image/jpeg,image/png,image/webp`

## Upload rules

- Photos are optional.
- Max 2 photos per lead.
- Original browser-selected image: max 5 MB.
- Browser compresses images before submission.
- Server rejects any final uploaded image over 1 MB.
- Typical compressed target: about 700 KB or less.

## Database table

The image file itself is stored in Supabase Storage. The `lead_photos` table only stores metadata:

- lead id
- storage bucket/path
- file name
- MIME type
- file size
- dimensions
- expiry/deletion timestamps

## Lifecycle

Uploaded photos default to 30-day expiry through `expires_at`.

The existing `/api/cron/process-due-leads` job now also deletes expired Storage objects and writes `deleted_at` on the matching `lead_photos` records.

## Privacy

The bucket is private. Admin and provider pages receive temporary signed URLs generated by backend service-role code.
